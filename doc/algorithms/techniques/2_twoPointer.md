# Two Pointer
## Overview
- Basic pattern for solving problems with linear structures
- Use two indicies to move through a structure to avoid nested loops and repetition
- Two Types:
    1. Pointers move in the same direction
    1. Pointers move in opposite directions

- Reduces number of iterations for the algorithm
- Track a relationship between two places
- Avoids extra space by not needing sets or maps

- Palindromes
- Reversals
- Merging Sorted Data
- "K" sized comparisons

### Same Direction
- Fast and Slow
    - Pointers move at different speeds (ie first pointer moves by 1 and second pointer moves by two)
        - Fast reaches the end while the other is halfway
            - found the middle
        - Fast equals the slow
            - found a cycle
    - Allows patterns to be detected
### Opposite Direction
- One starts at the begining, the other starts at the end, move inward
- Array is sorted, find pair or combination
- Comparing symmetric portions of a data structure (ie look for a palnidrome)
- avoid nested loops
- Move one of the pointers based upon the conditions of the program
