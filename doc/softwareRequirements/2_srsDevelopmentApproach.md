# SRS Development Approach
This document provides guidance on how to write an Software Requirements Specification (SRS) from scratch.

## SRS Structure
Start by creating a document who's structure aligns with the guidance provided by 1_srsStructure.txt.

## Generating SRS Content
Generally it is not feasible to just sit down and write an SRS with well-defined, testable requirements from start to finish.
In practice, it is best to start with a concept for the library or application that is being developed. Determine the high-level
goal or purpose of the software being developed and start developing the document using an iterative approach where each
iteration provides lower-level details that are closer to the desired functionality. This iterative approach is known as
"deriving requirements".
